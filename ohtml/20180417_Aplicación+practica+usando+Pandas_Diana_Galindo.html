UBER Movement¶

Descripción del proyecto¶

Para el siguiente proyecto se van a usar los datos agregados de tiempos en la ciudad de Bogot√° D.C. usando la plataforma de Uber. Uber Movement presenta datos agregados de viajes para los a√±os 2016, 2017 y 2018. A pesar de que para el diplomado se provee una parte del conjunto de datos, usted puede descargarlos usando su cuenta de Uber para ingresar a https://movement.uber.com. La plataforma presenta los datos de dos maneras: agregados trimestrales durante los tres a√±os y tiempos agregados de viaje desde un origen espec√≠fico hacia todos los destinos donde se hayan realizado viajes. Esta √∫ltima modalidad permite consultar agregados para rangos de tiempo que var√≠an entre 1 dia y 3 meses. A continuaci√≥n, se explica como se presentan los datos de cada formato: Opcional: Descargar desde el siguiente link: https://drive.google.com/open?id=1nhyqw7ZxmOsvgfdemlfc6lGNFWBPKFcz

Agregados trimestrales:¶

Son presentados en archivos .csv, donde cada registro corresponde a un origen y un destino. Entre par√©ntesis se ha colocado el nombre de cada archivo para su referencia (Sin extensi√≥n CSV)

  • Agregado horario para todos los d√≠as. (Origen, destino, hora del d√≠a, media, media geom√©trica, desviaci√≥n est√°ndar) (All-HourlyAggregate)
  • Agregado horario para los d√≠as entre semana. (Origen, destino, hora del d√≠a, media, media geom√©trica, desviaci√≥n est√°ndar) (OnlyWeekdays-HourlyAggregate)
  • Agregado horario para los fines de semana. (Origen, destino, hora del d√≠a, media, media geom√©trica, desviaci√≥n est√°ndar) (OnlyWeekends-HourlyAggregate)
  • Agregado mensual para todos los d√≠as. (Origen, destino, mes, media, media geom√©trica, desviaci√≥n est√°ndar) (All-MonthlyAggregate)
  • Agregado mensual para los d√≠as entre semana. (Origen, destino, mes, media, media geom√©trica, desviaci√≥n est√°ndar) (OnlyWeekdays-MonthlyAggregate)
  • Agregado mensual para los fines de semana. (Origen, destino, mes, media, media geom√©trica, desviaci√≥n est√°ndar) (OnlyWeekends-MonthlyAggregate)
  • Agregado por d√≠a de la semana. (Origen, destino, d√≠a de la semana, media, media geom√©trica, desviaci√≥n est√°ndar) (WeeklyAggregate)

Construya la serie de tiempo de la ruta SUPERCADE DE LA 30 - UNICENTRO durante Marzo y Abril de 2017, recolectando la informaci√≥n agregada de tiempo durante todo el d√≠a, durante las horas pico de la ma√±ana y durante las horas pico de la tarde. En total son 3 series de tiempo.

Primero defina una funci√≥n extract_time, que reciba una lista de DataFrames (Agregado diario, agregado hora pico ma√±ana o agregado hora pico tarde), el destino, y el tipo de dato agregado que quiere usar para la serie de tiempo (Promedio, valor m√≠nimo o valor m√°ximo). Como se muestra m√°s adelante, debe retornar una lista de 61 elementos que corresponden a los tiempos agregados hacia el destino solicitado. Debido a que los dataframes fueron creados por orden cronol√≥gico, no hay necesidad de volver a ordenarlos para obtener los datos de forma correcta. Tip: Use .values para extraer los valores en crudo de un DataFrame

In [233]:
def extract_time(list_dfs, dest, tipo='mean'):
    """
    list_dfs: lista, contiene la lista de DataFrames
    dest: int, contiene el MOVEMENT_ID del destino
    tipo: string, 'mean', 'min' o 'max'
    
    Regresa:
    time_list: lista, lista del tama√±o de list_dfs, 
               que contiene los tiempos agregados hacia 
               el destino dest, organizados por fecha.
    """
    
    time_list = []
    for df in list_dfs:
        if tipo == 'mean':
            value = df.loc[df[df.columns[2]] == dest, df.columns[5]].values # DEFINA SU C√ìDIGO AQU√ç. Use .values
        elif tipo == 'min':
            value = df.loc[df[df.columns[2]] == dest, df.columns[6]].values  # DEFINA SU C√ìDIGO AQU√ç
        else:
            value = df.loc[df[df.columns[2]] == dest, df.columns[7]].values # DEFINA SU C√ìDIGO AQU√ç
        value = value[0] if len(value) >= 1 else None
        time_list.append(value)
    return time_list
