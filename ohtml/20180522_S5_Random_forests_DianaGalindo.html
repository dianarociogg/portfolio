def plot_data(X, y):
    y_unique = np.unique(y)
    colors = pl.cm.rainbow(np.linspace(0.0, 1.0, y_unique.size))
    for this_y, color in zip(y_unique, colors):
        this_X = X[y == this_y]
        pl.scatter(this_X[:, 0], this_X[:, 1],  c=color,
                    alpha=0.5, edgecolor='k',
                    label="Class %s" % this_y)
    pl.legend(loc="best")
    pl.title("Data")
    
# Funcin para visualizar de la superficie de decisin de un clasificador
def plot_decision_region(X, pred_fun):
    min_x = np.min(X[:, 0])
    max_x = np.max(X[:, 0])
    min_y = np.min(X[:, 1])
    max_y = np.max(X[:, 1])
    min_x = min_x - (max_x - min_x) * 0.05
    max_x = max_x + (max_x - min_x) * 0.05
    min_y = min_y - (max_y - min_y) * 0.05
    max_y = max_y + (max_y - min_y) * 0.05
    x_vals = np.linspace(min_x, max_x, 100)
    y_vals = np.linspace(min_y, max_y, 100)
    XX, YY = np.meshgrid(x_vals, y_vals)
    grid_r, grid_c = XX.shape
    ZZ = np.zeros((grid_r, grid_c))
    for i in range(grid_r):
        for j in range(grid_c):
            ZZ[i, j] = pred_fun(XX[i, j], YY[i, j])
    pl.contourf(XX, YY, ZZ, 100, cmap = pl.cm.coolwarm, vmin= -1, vmax=2)
    pl.colorbar()
    pl.xlabel("x")
    pl.ylabel("y")
    
class MidpointNormalize(Normalize):

    def __init__(self, vmin=None, vmax=None, midpoint=None, clip=False):
        self.midpoint = midpoint
        Normalize.__init__(self, vmin, vmax, clip)

    def __call__(self, value, clip=None):
        x, y = [self.vmin, self.midpoint, self.vmax], [0, 0.5, 1]
        return np.ma.masked_array(np.interp(value, x, y))
    
def gen_pred_fun(clf):
    def pred_fun(x1, x2):
        x = np.array([[x1, x2]])
        return clf.predict(x)[0]
    return pred_fun

Problemas:

  • En algunos casos, las franteras de decisin generadas por el rbol de decisin (paralelas a los ejes) no son lo suficientemente flexibles para capturar la no linealidad del conjunto de datos.
  • Por otro lado, los rboles de clasificacin pueden crear reglas decisin muy espcficas que se ajustan demasiado a los datos de entrenamiento.
  • Tambien son sensibles a pequeas variaciones de los datos que pueden resultar en rboles totalmente distintos. Esto es un problema cuando se hace validacin cruzada de k-pliegues.

Cmo solucionar este problema?

Vamos a utilizar una estrategia conocida como clasificacin por comit o ensamble de clasificadores. Un ensamble de clasificadores combina diferentes algoritmos de aprendizaje para obtener un mejor desempeo predictivo. A continuacin, presentamos una forma de como construr un ensamble de rboles de clasificacin. Se entrenan diferentes rboles en diferentes subconjuntos de los datos de entrenamiento. Posteriormente cuando se va hacer una prediccin para un nuevo dato, se obtienen las predicciones de todos los rboles y se regresa aquella clase de mayor frecuencia predicha.

A la tcnica de la figura se le conoce como bagging. En el ejemplo siguiente vamos a utilizar la funcin BaggingClassifier la cual entrena un modelo usando esta estrategia. Los parmetros recibidos por la funcin son los siguientes:

  • base_estimator: Consiste en el estimador base que se va a entrenar sobre los diferentes subconjunto del conjunto de datos de entrenamiento.
  • n_estimators: Es el nmero de estimadores base que se van a usar para la estrategia de bagging.

Intuición detrás de RandomForests¶

Random Forest es una tcnica de ensamble que combina diferentes rboles de decisin. Estos rboles se entrenan en diferentes muestras del conjunto de datos, estos rboles pueden sobreajustarse, sin embargo la combinacin de sus predicciones resulta en un clasificador con menor sobreajuste. Para entender la idea de como se construye este algoritmo, suponga que entrenamos los siguientes rboles para determinar si una persona est enferma o sana:

Qu sucede si todos los rboles de clasificacin son iguales?

Cmo logro que cada rbol de clasificacin sea diferente?

Algoritmo básico Random Forest¶

A diferencia del rbol comn de decisin, los rboles de Random Forest se entrenan de una forma diferente. A continuacin, presentamos el algoritmo bsico de entrenamiento:

  1. Para cada rbol se realiza el siguiente procedimiento:
    • Se escoge una muestra con reemplazo de tamao $n$ del conjunto de entrenamiento.
    • Se seleccionan $m$ variables al azar de las variables disponibles
    • Se entrena un rbol sobre la muestra usando las $m$ variables repitiendo los siguientes pasos:
      • Se escoge la variable (y umbral) que genera la mejor particin
      • Se divide los datos en dos subconjuntos de acuerdo a la variable y el umbral
      • Si no se satisface un criterio de parada se aplica este procedimiento recursivamente sobre los subconjutos
  2. Una vez cada rbol ha sido entrenado, se genera el ensamble de rboles.

Implementación en Scikit-Learn¶

La implementacin en Scikit-Learn nos permite controlar los siguientes parmetros:

  • n_estimators: Nmero de rboles a entrenar
  • max_features: Nmero de variables $m$ al azar que se tienen en cuenta para la construccin de cada rbol.

Un nmero grande de rboles resulta en un buen desempeo a costas del costo computacional. Ambos parmetros deben ser explorados usando validacin cruzada. A continuacin, usaremos el conjunto de datos wine.

Optimización aleatoria de parámetros¶

A pesar de las ventajas que ofrece GridSearchCV sobre la exploracin sistemtica de los hiperparmetros de un modelo, puede gastar un tiempo considerable en esta exploracin. Scikit-Learn permite hacer tambin una exploracin aleatoria de los parmetros, la cual se ha demostrado empiricamente que es ms eficiente que optimizar los parmetros usando una malla con GridSearchCV (Referencia). RandomizedSearchCV implementa una bsqueda aleatoria sobre los parmetros. El rango de exploracin de los parmetros se puede especificar de la siguiente manera:

  • Usando una lista:
    • "criterion": ["gini", "entropy"]
  • Usando una distribucin discreta uniforme:
    • "n_estimators": randint(4, 2048)
    • randint proviene de scipy.stats. Este genera una distribucin discreta entre 4 y 2048.
  • Usando una distribucin uniforme:
    • "max_features": uniform()
    • uniform proviene de scipy.stats. Este genera una distribucin uniforme entre 0 y 1.

RandomizedSearchCV puede recibir tanto listas de elementos como distribuciones de probabilidad, las cuales deben ser especificadas usando scipy.stats. A continuacin, creamos nuestro estimador y definimos la distribucin de parmetros:
